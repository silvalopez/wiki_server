  • ¿Que comandos básicos debo tener en cuenta para ingresar al servidor?

    • cd: Para navegar por los archivos y directorios de Linux. Requiere la ruta completa o el nombre del directorio, según el directorio de trabajo actual en el que se encuentre.
    • ls: El comando ls se usa para ver el contenido de un directorio.
      • ls -a: muestra los archivos ocultos
      • ls -lh: muestra los archivos con información de la fecha
      • ls -al: lista todos los archivos y directorios con información detallada como permisos, tamaño, propietario, etc.
    • pwd: Comando para averiguar la ruta del directorio de trabajo actual (carpeta) en el que se encuentra.
    • cp: Para copiar archivos del directorio actual a un directorio diferente.
    • mv: El uso principal del comando mv es mover archivos, aunque también se puede usar para cambiar el nombre de los archivos.
    • mkdir: Para crear un nuevo directorio
    • rm: Se usa para eliminar archivos y directorios y el contenido dentro de ellos (rm -r)
    • chmod: Es otro comando de Linux, que se utiliza para cambiar los permisos de lectura, escritura y ejecución de archivos y directorios. chmod 777 FileOrFolder lo utilizamos para dar permisos de todo
    • df -lh: Ver espacio en el disco
    • htop: Es un visor de procesos y un administrador de procesos del monitor del sistema.
    • crontab -e: para ingresar a la lista de crones
    • screen: Sirve para crear “terminales virtuales”. Con -S para crear una nueva sesión, con -R para recuperar una
    • ipython3, ipython: Para abrir una consola de python

  • Revisión espacio y procesos

  • ¿Cómo puedo revisar cuánta RAM hay disponible y en general el consumo de los procesos del servidor?

    Con htop.htop es un visor de procesos y un administrador de procesos de Ubuntu. Con este visor se muestra una lista actualizada con la frecuencia de los procesos que se ejecutan, normalmente ordenados por la cantidad de uso de la CPU, o por RAM. Cuando le damos el comando htop en el servidor, se abre la siguiente instancia: En la parte superior, donde dice Mem, se muestra la cantidad de memoria RAM que se esta utilizando, y en la parte de abajo se encuentran listados todos los procesos que están corriendo, con el porcentaje de RAM y CPU que están empleando. En este link se explica con mayor detalle los componentes de este visor. Para salir de este visor se presiona f10.

  • ¿Cómo puedo revisar cuánto espacio hay disponible?

    Con el comando df -lh puedo ver todas las particiones del sistema, su espacio disponible y la ubicación. El servidor tiene dos particiones de almacenamiento principales, y son aquellas señaladas con resaltador amarillo en la imagen. Cada una tiene una capacidad de 3.6T. Tal y como se puede evidenciar, actualmente la partición con más espacio disponible esta ubicada en /mnt/home2. Cuando se necesiten realizar actividades en el servidor que demanden mucho espacio, se recomienda ir revisando con este comando cuánto se esta consumiendo, y así identificar si necesito trasladar mis datos o mi proceso, a la partición con más espacio disponible.

  • Procesos activos en el servidor

  • ECMWF

    En el servidor están los archivos desde 2020-01-07.

    • Cada archivo contiene 51 ensambles. Cada ensamble parte de datos iniciales ligeramente perturbados y evoluciona un poco diferente de los otros miembros del conjunto para dar una gama de posibles resultados de pronóstico. La variación vista en los pronósticos de cada ensamble da una indicación de la predictibilidad de la atmósfera.
    • Formato de los datos: GRIB
    • Proyección: Cylindrical Equidistant
    • Tamaño de píxel = 0.2°x 0.2°
    • Hay 3 horizontes diferentes de pronóstico disponibles:
      • A 6 días: Stream X - con una resolución temporal de 1 hora (de 0 a 90 horas pronosticadas en adelante) y de 3 horas (de 90 a 144 horas pronosticadas en adelante).
        La variable de precipitación pronosticada está en magnitud de lámina de agua [m].
        Se actualizan 3 veces al día: las corridas de las 0000 UTC, 0600 UTC y 1200 UTC
      • A 15 días: Stream E - con una resolución temporal de 6 horas.
        La variable de precipitación pronosticada está en magnitud de lámina de agua [m].
        Se actualizan 2 veces al día: las corridas de las 0000 UTC y 1200 UTC
      • A 6 meses: Stream L - con una resolución temporal mensual.
        La variable de precipitación pronosticada está en intensidad [m/s].
        Se actualizan el día 5 de cada mes a las 0700hrs local.
    A continuación se explica de manera más detallada la convención de los nombres de los archivos y la configuración en el servidor para el procesamiento de los datos operacionales:  

    Convención nombre archivos ECMWF

    La siguiente convención de nombre de archivo de difusión se utiliza para la transmisión de productos de difusión de ECMWF:

    ccSMMDDHHIImmddhhiiE

    Donde:
    • cc es el nombre del flujo de difusión
    • S es indicador de flujo de datos de difusión
    • MMDDHHII es el mes, día, hora y minuto en que se basan los productos.
    • mmddhhii es el mes, día, hora y minuto en que los productos son válidos
    • mmddhhii está configurado como "______" para los productos del Sistema de Pronóstico Estacional
    • hhii está configurado como "______" para los productos de pronóstico mensual
    • ii se establece en 1 para el tiempo de pronóstico de alta resolución paso cero.
    • E es el número de versión del experimento (como EXPVER en MARS, normalmente 1)
    En el caso de los archivos que estamos recibiendo, de acuerdo al nombre del archivo, se han identificado 3 tipos:
    • Pronóstico con horizonte de tiempo de 6 días:
      cc = 'G1'
      S = 'X'
      De esta manera, todos estos archivos siempre comienzan con G1X: G1XMMDDHHIImmddhhi1
    • Pronóstico con horizonte de tiempo de 15 días:
      cc = 'G1'
      S = 'E'
      De esta manera, todos estos archivos siempre comienzan con G1E: G1EMMDDHHIImmddhhi1
    • Pronóstico con horizonte de tiempo de 7 meses:
      cc = 'G1'
      S = 'L'
      De esta manera, todos estos archivos siempre comienzan con G1L: G1LMMDDHHIImm ______1
